**Status:** ✅ SHIPPED 2026-07-07

**Total Plans:** 20

## Overview

The initial release of Looking For Group (LFG): a free, self-hostable Doodle "Group Poll" clone for scheduling. An organizer proposes candidate dates, shares an unguessable participant link (optionally emailed), and each participant marks every date Available / If-need-be / Not available with no account. A live admin dashboard grids everyone's answers, highlights the best day(s), and lets the organizer "Book it" — closing the poll and emailing all voters. Runs $0 on local Docker Postgres and Vercel free tier + Neon, with live Gmail SMTP email in production. All 30 v1 requirements shipped and verified.

## Milestone Summary

**Decimal Phases:** None.

**Key Decisions:**

- Three-token access model — separate `adminUrlId`, `participantUrlId`, and per-participant `editToken`, none derivable from another (least-privilege sharing; IDOR defense)
- Crypto-random non-enumerable nanoid IDs for all public identifiers; no auto-increment integers in URLs
- Postgres everywhere — Docker local + Neon prod; Neon over Supabase (no 7-day pause)
- Date-only slots stored as Postgres DATE and parsed without `new Date()` (no timezone drift — PLAT-04)
- Dual-driver Drizzle client typed as a single concrete driver type (collapses overloaded query signatures across neon-http/node-postgres)
- Single env-switched `sendEmail()` seam (none/smtp/resend); email fully optional ($0 default, graceful degradation — MAIL-03/D-02)
- Gmail SMTP in prod with `EMAIL_FROM = SMTP_USER` (self-aligns SPF/DKIM/DMARC on a free tier; SMTP2GO single-sender pre-wired fallback)
- WCAG role=radiogroup/radio matrix for the vote grid, both responsive layers in DOM with display:none a11y isolation
- Account-free organizer identity via httpOnly `lfg_organizer` cookie + nullable indexed `polls.organizer_id` (same-browser poll grouping without auth)

**Issues Resolved:**

- Missing `Secure` flag on the `lfg_edit` cookie — caught by the production smoke test, not unit tests
- Desktop horizontal-overflow bug in the redesign (wide results table needed `min-w-0` on the grid track) — caught by full-page prod screenshot verification
- `decision-coverage-plan` gate false-negative on phase-prefixed decision IDs — all IDs renamed to bare `D-NN`

**Issues Deferred (see STATE.md Deferred Items):**
