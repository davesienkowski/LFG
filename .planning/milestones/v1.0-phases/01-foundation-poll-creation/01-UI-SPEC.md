# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Foundation & Poll Creation.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

## Design System

| Property | Value | Source |
|----------|-------|--------|

| Icon library | lucide-react (bundled with shadcn/ui) | shadcn standard |

## Spacing Scale

Standard 8-point scale. All values are multiples of 4.

| Token | Value | Tailwind class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `gap-1` / `p-1` | Icon gap, inline chip padding |
| sm | 8px | `gap-2` / `p-2` | Compact element spacing, icon-label gap |
| md | 16px | `gap-4` / `p-4` | Default field spacing, card content padding |
| lg | 24px | `gap-6` / `p-6` | Section padding, card padding |
| xl | 32px | `gap-8` / `p-8` | Major section breaks (e.g. form sections) |
| 2xl | 48px | `gap-12` / `p-12` | Page-level vertical padding |
| 3xl | 64px | `gap-16` / `p-16` | Not used in Phase 1 |

**Exceptions:**

- `<input type="date">` and `<input type="time">`: minimum height 44px for touch target compliance (use `h-11` / `min-h-[44px]`). Standard shadcn `input` component height is 36px — override with `h-11` on these specific inputs.
- Date row (date + optional time + remove button): horizontal flex with `gap-2` between the date input and time input, `gap-4` between the time input and the remove button.

## Typography

Four sizes, **two weights** (400 regular, 600 semibold). Do not introduce additional sizes or weights in Phase 1. (Label uses 600 semibold at 14px — visually distinct from 16px regular body, so no medium weight is needed.)

| Role | Size | Weight | Line Height | Tailwind | Usage |
|------|------|--------|-------------|---------|-------|
| Display | 30px | 600 (semibold) | 1.1 | `text-3xl font-semibold leading-tight` | Page headings: "Create a poll", poll title on admin page |
| Heading | 24px | 600 (semibold) | 1.2 | `text-2xl font-semibold leading-snug` | Section headings: "Candidate dates", "Share your poll" |
| Body | 16px | 400 (regular) | 1.5 | `text-base font-normal leading-relaxed` | Field helper text, description text, placeholder content |
| Label | 14px | 600 (semibold) | 1.5 | `text-sm font-semibold leading-normal` | Field labels, link card labels, button text, badge text |

Font stack: `Inter, system-ui, -apple-system, sans-serif` loaded via `next/font/google`.

## Color

Using shadcn/ui CSS variable system (zinc base, light mode only in Phase 1).

| Role | CSS Variable | Approximate Hex | Usage |
|------|-------------|-----------------|-------|
| Dominant (60%) | `--background` | `hsl(0 0% 100%)` white | Page background, form background |
| Secondary (30%) | `--card` / `--muted` | `hsl(240 4.8% 95.9%)` zinc-100 | Card backgrounds, muted section backgrounds |
| Accent (10%) | `--primary` | `hsl(240 5.9% 10%)` zinc-900 | Reserved: Create poll CTA button only |
| Destructive | `--destructive` | `hsl(0 84.2% 60.2%)` red-500 | Reserved: remove-date row button only |

**Accent reserved for:**
- The "Create poll" submit button (and its loading state "Creating...")
- No other elements use `--primary` in Phase 1

**Destructive reserved for:**
- The "Remove" icon button on date rows (text/icon uses `text-destructive` on hover)
- Not used for confirmation dialogs (none in Phase 1)

**Semantic color additions for Phase 1:**
- Past-date warning: `text-amber-600` / `bg-amber-50` — soft warning badge below a date input when the entered date is in the past
- Validation error: `text-destructive` — inline error messages
- Copy-success feedback: `text-emerald-600` — "Copied!" transient state on copy buttons
- Admin link "keep private" badge: `bg-amber-50 text-amber-700 border border-amber-200` — distinguishes admin link card visually

No dark mode in Phase 1.

## Surfaces

### Surface 1: Poll Creation Form `/`

**Layout:** Centered single-column, max-width 640px (`max-w-2xl mx-auto`), vertical padding 48px top/bottom.

**Focal point:** The "Create a poll" h1 Display heading draws the eye first; the zinc-900 "Create poll" CTA button is the secondary focal anchor (the only accent-colored element on the page). On the admin page, the poll title (Display) is primary and the amber-bordered Admin link card is the secondary anchor; on the participant shell, the poll title is primary and the muted "Voting isn't available yet" banner is secondary.

**Field order (top to bottom):**

1. Page heading: "Create a poll" (Display / 30px / semibold)
2. **Title** field — required
   - Label: "Poll title"
   - Input: `<input type="text" name="title" />` (shadcn `Input`)
   - Placeholder: "e.g. D&D Session — July"
   - Max length: 200 characters (enforce server-side via Zod; show counter at 180+)
3. **Description** field — optional
   - Label: "Description (optional)"
   - Input: `<textarea name="description" rows={3} />` (shadcn `Textarea`)
   - Placeholder: "Add notes for your group — location details, what to bring, etc."
   - Max length: 2000 characters (per SPEC requirement 2 / POLL-02)
4. **Location** field — optional
   - Label: "Location (optional)"
   - Input: `<input type="text" name="location" />` (shadcn `Input`)
   - Placeholder: "e.g. Dave's place, Discord, Tavern on Main St"
   - Max length: 200 characters (per SPEC requirement 2 / POLL-02)
5. **Candidate dates** section
   - Section heading: "Candidate dates" (Heading / 24px / semibold)
   - One date row rendered on initial load (the minimum required row)
   - "Add date" button (variant=`outline`, icon=`Plus`, label="Add date") — appended below the date list
6. **Submit button** — "Create poll" (variant=`default` using `--primary`, full-width on mobile, auto-width on desktop)

**Date row structure (each row):**

```
[date input 44px]   [time input 44px — labeled "Start time (optional)"]   [X remove button]
```

- Date input: `<input type="date" />`, `h-11`, styled as shadcn `Input`
- Time input: `<input type="time" />`, `h-11`, styled as shadcn `Input`, always visible alongside date input (not behind a toggle)
- Remove button: `variant="ghost"` + `size="icon"` + `Trash2` lucide icon, `text-muted-foreground hover:text-destructive`
- Minimum one row at all times: when only one row exists, the remove button is hidden (not disabled)

### Surface 2: Admin Page `/a/[adminUrlId]`

**Layout:** Centered single-column, max-width 640px (`max-w-2xl mx-auto`), vertical padding 48px.

**Sections (top to bottom):**

1. Page heading: poll title from DB (Display / 30px / semibold)
2. Poll summary (if description or location present):
   - Description: body text, up to 3 lines visible (no truncation)
   - Location: body text with `MapPin` lucide icon inline
3. **Candidate dates** list:
   - Each date rendered as a readable string (e.g. "Saturday, July 12" for date-only; "Saturday, July 12 at 2:00 PM" for date+time)
   - Format: string-based, never via `new Date()` constructor (per D-11)
   - Ordered chronologically (DB returns `ORDER BY date, start_time`)
4. Section heading: "Share your poll" (Heading / 24px / semibold)
5. **Participant link card** (shadcn `Card`):
   - Label: "Participant link" (Label / 14px / semibold)
   - Helper: "Share this link with your group" (Body / 16px)
   - URL displayed as truncated monospace text (`font-mono text-sm truncate`)
   - "Copy link" button (variant=`outline`, icon=`Copy` → `Check` on success)
6. **Admin link card** (shadcn `Card`, with amber visual distinction):
   - Label: "Admin link" (Label / 14px / semibold) + "Keep private" badge (`bg-amber-50 text-amber-700 border border-amber-200 rounded-full px-2 py-0.5 text-xs font-semibold`)
   - Helper: "Do not share this link. It grants full management access to this poll." (Body / 16px, `text-muted-foreground`)
   - URL displayed as truncated monospace text
   - "Copy link" button (variant=`outline`, icon=`Copy` → `Check` on success)
   - Card border: `border-amber-200` to visually distinguish from participant link card

### Surface 3: Participant Route Shell `/p/[participantUrlId]`

**Layout:** Centered single-column, max-width 640px (`max-w-2xl mx-auto`), vertical padding 48px.

Phase 1 renders a resolved placeholder — no voting UI.

**Sections (top to bottom):**

1. Poll title (Display / 30px / semibold)
2. Poll summary (description + location if present, same as admin page summary)
3. Candidate dates list (same format as admin page, chronological)
4. Placeholder banner:
   - Container: `rounded-lg border bg-muted p-6 text-center`
   - Heading: "Voting isn't available yet" (Heading / 24px / semibold)
   - Body: "The organizer is still setting up this poll. Check back soon." (Body / 16px / `text-muted-foreground`)

---

### Surface 4: 404 / Not Found

Applies to: invalid `[adminUrlId]` → `/a/[adminUrlId]`; invalid `[participantUrlId]` → `/p/[participantUrlId]`.

Implemented via Next.js App Router `not-found.tsx` per-route segment or global.

**Content:**

- Heading: "Poll not found" (Display / 30px / semibold)
- Body: "This link may be invalid or the poll may no longer exist." (Body / 16px / `text-muted-foreground`)
- No retry button. No home link required (MVP). Page is otherwise blank.

## Interaction States

### Form — Validation Errors

Displayed inline immediately below the relevant field after the Server Action returns `{ errors }`. Each error uses `role="alert"` and `text-destructive text-sm` styling.

| Trigger | Field | Error Copy |
|---------|-------|-----------|
| Submit with empty title | Title input | "Poll title is required" |
| Submit with whitespace-only title | Title input | "Poll title is required" |
| Submit with title > 200 chars | Title input | "Title must be 200 characters or fewer" |
| Submit with description > 2000 chars | Description textarea | "Description must be 2,000 characters or fewer" |
| Submit with location > 200 chars | Location input | "Location must be 200 characters or fewer" |
| Submit with zero date rows | Candidate dates section | "Add at least one candidate date" |
| Submit with invalid date format | Affected date row | "Enter a valid date" |

### Form — Past Date Warning

Not a blocking error — a soft warning shown **as the user types/selects** (client-side only, via `onChange` on the date input).

- Rendered: amber badge below the date input — `text-amber-600 text-sm` — "This date is in the past"
- The form can still be submitted with a past date (no server-side rejection of past dates)

### Form — Submitting / Pending State

Triggered when the Server Action is in-flight (`isPending` from `useActionState`).

- Submit button: disabled, text changes to "Creating..." (no spinner required for MVP)
- All form inputs: `disabled` attribute set
- Date add/remove buttons: disabled

### Admin Page — Copy Link Feedback

Triggered when the user clicks "Copy link" on either share link card.

- Button icon changes from `Copy` to `Check` (lucide icons)
- Button text changes from "Copy link" to "Copied!"
- Button variant stays `outline`
- After 2000ms, reverts to `Copy` / "Copy link" state
- Uses `navigator.clipboard.writeText()` inside a `try/catch`; on failure, silently log (no visible error for MVP)

### Date Row — Add / Remove

- **Add date**: clicking "Add date" appends a new empty date row at the bottom of the list. New row receives focus on the date input (use `useEffect` + `ref` to focus).
- **Remove date**: clicking the remove icon button removes that row from the client-side list. If only one row remains, the remove button is not rendered (hidden via conditional render, not `display:none`). Removing a row does not trigger validation immediately.

## Accessibility Requirements

- All form fields have associated `<label>` elements via `htmlFor` / `id` pairing (shadcn `Label` + `Input`)
- Validation error messages use `role="alert"` so screen readers announce them immediately after form submission
- The `aria-describedby` attribute links each input to its error element when an error is present
- Copy buttons use `aria-label="Copy participant link"` / `aria-label="Copy admin link"` (not just "Copy link" — unique per button on the page)
- Date inputs for multiple rows: each has a screen-reader `<label>` with a numbered description ("Date 1", "Date 2", etc.) using `sr-only`
- Time inputs: labeled as "Start time for date {N} (optional)" via `sr-only`
- Remove buttons: icon-only visually, `aria-label="Remove date {N}"` and `sr-only` span text "Remove date"
- Minimum touch target 44×44px on interactive elements (date inputs, time inputs, remove buttons)
- Focus order: title → description → location → date 1 → time 1 → date 2 → time 2 → ... → add date button → create poll button
- No color-only communication: the "Keep private" admin link warning uses both amber color AND the text badge label

## Prohibitions (UI — prohibition-probe)

**Probe policy:** Deterministic edge-engine SKIPPED for this UI-SPEC — the contract introduces no new data shapes (dates/tokens were fully edge-probed at SPEC). Prohibition-probe run over the NEW UI behaviors only. Upstream-covered items (admin-link leak = SPEC P2; `new Date()` TZ shift = SPEC P3) are NOT re-probed here.

| Prohibition (must-NOT statement) | Surface | Status | Verification |
|----------------------------------|---------|--------|--------------|
| MUST NOT display the admin link without its "Keep private" badge AND the do-not-share warning text (not color alone) | Admin page `/a/[adminUrlId]` | resolved | test — assert admin link card contains the "Keep private" badge text and the warning copy; color is not the sole signal |
| MUST NOT show the "Copied!" success state when `navigator.clipboard.writeText()` rejects or throws | Admin page copy buttons | resolved | test — mock a clipboard rejection and assert the button does NOT enter the "Copied!"/`Check` state |

These become `must_haves` for the planner / verifier (negative acceptance criteria).
