<objective>
Deliver the first full vertical slice: an organizer fills the creation form at `/`, submits, the `createPoll` server action validates and persists the poll + options with two independent unguessable tokens, and the organizer is redirected to `/a/[adminUrlId]` showing both share links. Add the participant shell `/p/[participantUrlId]` (resolves a poll, voting deferred to Phase 2) and a 404 for invalid/garbage tokens.

Purpose: This is the phase's core user-facing capability (POLL-01..04, LINK-01..03), built on the schema, tokens, date helper, and dev environment from 01-01. The admin/participant token separation and date handling locked here are load-bearing for every later phase.
Output: A working create -> admin flow proven locally in Docker Desktop, with all SPEC acceptance criteria, edge cases, and prohibitions satisfied.

User story:
**As a** poll organizer (D&D host), **I want to** create a scheduling poll with a title, optional details, and candidate dates and receive two distinct share links, **so that** I can send my group a voting link while keeping a private management link for myself.
</objective>
