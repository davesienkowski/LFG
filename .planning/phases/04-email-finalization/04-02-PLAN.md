<objective>
Deliver the finalization vertical slice: the additive `winning_option_id` schema
change (with the mandatory local migration gate), a `closePoll` "Book it" server
action that closes voting in a single UPDATE and best-effort-notifies every voter
with a stored email, and the Book-it admin UI (a candidate-date picker with the
computed best day pre-selected, a two-step confirm disclosure, and a "Poll
finalized" state once closed).

Purpose: Let the organizer commit to a winning date, close voting (reusing Phase
2's server-enforced read-only path — no new enforcement code), and communicate the
decision by email — the phase's second and final user-facing slice.

Output: `polls.winning_option_id`, `src/lib/actions/close-poll.ts`, new read
helpers in `queries.ts`, `src/components/book-it-control.tsx`, and the Book-it /
finalized sections on the admin page. Finalization emails reuse Plan 04-01's
`sendEmail()` seam + `renderFinalizationEmail` template (this plan depends on 04-01).
</objective>

<assumption_delta_decision>
Primary noun = `poll`. Decision = `no-change`. Finalization adds an additive,
nullable `winning_option_id` FK on `polls` (references the chosen date option) plus
reuse of the existing `status` open→closed transition. No new top-level entity; no
change to the poll's primary key or identity. Rationale: "no winner yet" is an
intrinsic, legitimate state of an open poll, not accidental debt — a nullable FK
correctly models it. This is NOT an add-alongside that contradicts a generalized
intent; there is no competing singular assumption being reintroduced. (Recorded
verbatim per the advisory assumption-delta finding; settled — do not re-open with
new options.)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Organizer browser → closePoll (Server Action) | Untrusted adminUrlId + winningOptionId cross here; admin authority is the nanoid admin token |
| closePoll → polls UPDATE | A finalize write flips status + records the winner; must be admin-gated and option-validated |
| closePoll after() → SMTP/Resend | Poll-derived subject/body + voter addresses cross into the mail transport |
| Finalization email → voter inbox | Chosen date + event details rendered into HTML delivered to third parties |

| T-04-08 | Information Disclosure | getVoterEmailsForPoll / finalization template | mitigate | Query selects only name/email scoped to this poll; never edit_token/admin_url_id.
