<objective>
Build the read + send backend for respondent tracking and nudging: an admin-only
query that labels each invited email responded / not-responded (RESP-01 data
source), a distinct reminder email template, and the `nudgeNonRespondents` server
action that re-emails only the CURRENT non-respondents (RESP-02) — with all three
load-bearing correctness guards from the UI Prohibition Probe implemented
server-side.

Purpose: The UI plan (07-03) renders these; the nudge action is security-critical
(admin-token authorization, server-side recompute, closed-poll re-check,
no-admin-URL email), so it gets its own focused plan.

Output: `getInvitationTrackingForPoll`, `renderReminderEmail`,
`nudgeNonRespondents` — all tested.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin browser -> nudgeNonRespondents | Only `adminUrlId` crosses; the admin token is the authorization. A client-supplied recipient list or poll id is untrusted and ignored |
| server action -> sendEmail seam | Reminder HTML + a single string recipient per send; env-switched transport |
| invitations/participants read -> admin surface only | Invitation emails are PII; the tracking read is admin-only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Information Disclosure | getInvitationTrackingForPoll | mitigate | Admin-only read; never called by participant routes; non-vacuous CANARY leak test in 07-03 proves emails never reach a participant payload |
| T-07-02 | Spoofing / Elevation of Privilege | nudgeNonRespondents | mitigate | Re-derive poll from admin token (notFound on miss); recompute recipients server-side; form carries only adminUrlId; ignore any client recipient list |
| T-07-03 | Tampering (business logic) | nudgeNonRespondents | mitigate | Re-check `poll.status !== "closed"` server-side before any send, independent of the client-side hide |
| T-07-04 | Information Disclosure | renderReminderEmail | mitigate | Participant-recipient template; accepts only participantUrl; asserted to contain no `/a/` admin path (T-04-02) |
| T-07-10 | Elevation of Privilege (cross-tenant) | getInvitationTrackingForPoll responded match | mitigate | EXISTS correlated to invitations.poll_id; cross-poll isolation test proves a same-email voter on another poll never flips responded |
| T-07-SC | Tampering (supply chain) | npm installs | accept | No new dependencies — reuses sendEmail (nodemailer/resend) and drizzle already vetted in v1.0 |
</threat_model>
