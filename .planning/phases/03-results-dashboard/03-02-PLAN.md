<objective>
Render the Results Dashboard: a `"use client"` `ResultsGrid` island that turns plan 03-01's server-computed data into a semantic, accessible participants × dates table with three-state cells, per-date tallies, best-day highlighting, and a client-only date/status filter — mounted on the admin page below the existing Share section.

Purpose: This is the organizer-facing decision surface (DASH-01..05). It reuses the exact three-state vocabulary participants saw (D3-05) and never leaks email/tokens. All interactivity is client-side over already-fetched props (D3-06) — zero network round-trips on filter, and the filter derives its rows purely during render to avoid the React 19 double-invocation hazard Phase 2 already hit.
Output: `src/components/results-grid.tsx` + `results-grid.test.tsx` (new), and additive changes to `src/app/a/[adminUrlId]/page.tsx` + `page.test.ts`.
</objective>
