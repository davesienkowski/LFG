<objective>
Deliver DEAD-01: an optional voting deadline that lazily closes a poll to
further voting. The organizer sets/updates/clears it from the admin view; once
it passes, every vote gate (both participant pages, both vote actions, the
vote-form readOnly branch) derives "closed" from `isVotingOpen(poll, now)` — no
cron, no status write on read. The deadline is a distinct concept from
"Book it": a deadline-passed poll is voting-closed but NOT booked and claims no
winner.

Purpose: Success Criteria 1–3 for Phase 8. The load-bearing invariant is that
the ONE lazy-close rule (plan 01's isVotingOpen) is applied at EVERY gate
(LOCKED constraint 2), enforced server-side so a stale open form cannot vote
after the deadline (LOCKED constraint 4), while Booked stays keyed on
status === 'closed' (LOCKED constraint 3).

Output: setDeadline action; isVotingOpen wired at all vote gates; participant
read-only + distinct deadline-passed copy; the admin "Voting deadline" card +
header pill.
</objective>
