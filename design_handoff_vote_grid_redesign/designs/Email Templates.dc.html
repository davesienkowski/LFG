Ports 1:1 — already shipped

These three mirror renderInviteEmail / renderConfirmationEmail / renderFinalizationEmail. The only merge note: the app UI moved to the emerald/amber Matrix palette, but the emails deliberately stay neutral (hex-approximated tokens, no state colors) — correct, since a transactional mail has no vote cells to color. Keep them as-is.

Finalization now carries two neutral add-to-calendar buttons above the fallback link — a calendar.google.com render URL and the hosted /p/…/event.ics (participant token only, never /a/). Both are omitted cleanly if the calendar build fails.

Proposed tweak (needs a small code change): the two buttons now differ by color — Google's blue #1a73e8 vs neutral #171717 for Apple / Outlook — so a reader can tell them apart at a glance. Real email can't use SVG/PNG icons (Gmail strips them; the template forbids images), so color is the only reliable signal. This asks calLink() in templates.ts to take a per-provider background arg.
