Create · Thanks · Edit · Results / Admin

The four supporting screens, each built from the shipped shadcn primitives (Card, Button, Input, Textarea) and the verbatim tokens. The admin surface carries the two Phase-4 additions — the Invite by email card and the one-way Book it finalize control (two-step confirm, destructive action spatially separated from the safe dismiss). Results reuse the same three-state chip vocabulary as the vote grid, so the participant and organizer surfaces stay one mental model.

On a closed poll the Invite-by-email card and the Book-it picker are both hidden; the Results grid stays visible (read-only). The title gains the emerald Booked pill.

Mobile · the primary product

Every screen is one centered column at any width. Create and Edit adopt the vote screen's fixed-height scroll + pinned primary action so a long form never buries the button. Admin scrolls naturally — it has several contextual actions (copy, send, book), so nothing is pinned; the results table scrolls horizontally with an edge fade.
